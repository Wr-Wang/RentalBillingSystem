# 通用批量导入 — 需求文档

## 1. 背景

系统有多处需要批量导入数据且经审批后才能生效：房源导入、合同导入、租客导入等。需要一个**通用导入审批框架**，统一处理公共流程，同时允许各场景差异化的校验和执行逻辑。

## 2. 设计原则

- **数据库不允许存 JSON 原始数据** — 所有字段必须有明确定义的列和类型
- **通用与独立分离** — 各导入类型共用的字段放入基础表，差异字段放入类型专属扩展表
- **审批驱动** — 所有导入均需审批通过后才创建业务数据
- **重导兼容** — 同一份文件可修正后重新导入，已创建的行自动跳过

## 3. 适用范围

| 导入类型 | 代码标识 | 关联审批类型 | 优先级 |
|----------|----------|-------------|--------|
| 房源导入 | `HousingUnit` | 批量导入房屋（三级审批） | **P0 当前实现** |
| 合同导入 | `Contract` | 新建合同（三级审批） | P1 后续 |
| 租客导入 | `Tenant` | 待定 | P2 后续 |

## 4. 通用流程

```
① 下载模板 → ② 填写数据 → ③ 上传 Excel → ④ 系统校验
  → ⑤ 有效行暂存 → ⑥ 提交审批 → ⑦ 审批通过 → ⑧ 创建业务实体
```

- 步骤 ④⑤⑥ 为**同步**（用户等待结果）
- 步骤 ⑦⑧ 为**异步**（审批人操作后自动执行）

## 5. 通用功能

### 5.1 模板下载 & Excel 解析

- 每种导入类型有独立模板
- 必填列标红 `*`，列头文字加粗
- 顶部黄色提示区注明：
  - ⚠ 房型列注明：「请根据系统最新房型填入，当前可用房型见 Sheet2」
  - ⚠ 金额不需要填写，系统根据房型+楼层自动计算
  - ⚠ 红色 `*` 列为必填，不可留空
- **模板列级别强制拦截**：必填列设置 Excel 数据验证（Data Validation），拒绝空值输入
  - **房型列设置下拉列表**（引用 Sheet2 的房型数据），下载模板的人直接从下拉菜单中选择，**禁止手工输入**
  - 朝向列设置下拉列表（东/南/西/北/南北通透/东南/西南/东北/西北）
  - 面积列限制只能输入正数
- Sheet2 提供系统参考数据：
  - 当前公司所有可用房型列表（从系统实时加载）
  - 当前公司定价标准参考（房型×楼层级别×租金）
  - 预设朝向列表

### 5.2 逐行校验（格式检查）

- 必填列空值、枚举值非法、引用数据不存在、唯一性冲突 → 标记该行失败
- 校验通过的行进入待审批状态，失败的行标记原因
- 红色 `#f56c6c`：必填缺失/引用不存在；橙色 `#e6a23c`：数值越界；黄色 `#f0c040`：唯一性冲突

### 5.3 导入结果展示

提交导入后，无论成功还是失败，都要立即返回明确的导入结果页：

**统计卡片**（顶部）：
| 指标 | 说明 |
|------|------|
| 总行数 | Excel 中待导入的总行数 |
| 成功提交 | 格式校验通过、已进入审批流程的行数 |
| 失败行数 | 格式校验未通过、被拦截的行数 |

**Tab 切换列表**：
- **成功列表**：展示已提交审批的行明细（座楼/楼层/房号/房型/面积/自动计算租金/租金警告）
- **失败列表**：展示被拦截的行明细（座楼/楼层/房号/错误码/错误原因/**修正建议**），失败原因用颜色标记，并提供如何修改的说明

**失败列表列定义**：

| 列 | 说明 |
|----|------|
| 行号 | Excel 中的原始行号 |
| 座楼/楼层/房号 | 该行数据 |
| 错误码 | 程序标识，如 `ERR_REQUIRED_BUILDING` |
| ❌ 失败原因 | 具体错误描述，**颜色标记严重程度** |
| 💡 修正建议 | 告诉用户如何修改，如「请在"座楼名称"列填写座楼名称，如：A栋」 |

**失败原因颜色标记 + 对应修正建议**：

| Severity | 颜色 | 适用场景 | 修正建议示例 |
|----------|------|----------|-------------|
| 🔴 error | `#f56c6c` | 必填缺失、引用数据不存在 | 「请在"座楼名称"列填写座楼名称，如：A栋」 |
| 🟠 error | `#e6a23c` | 数值越界、枚举不合法 | 「请在"朝向"列从下拉列表中选择，可选值：东、南、西、北、南北通透」 |
| 🟡 error | `#f0c040` | 唯一性冲突 | 「该房源已存在系统中，如需修改请删除对应行后重导」 |
| ⚠ warning | `#909399` | 业务警告（定价匹配不到） | 「该房型在对应楼层无定价标准，请联系管理员设置后重新导入」 |

**操作按钮**：
- 「返回列表」：回到房源管理页
- 「继续导入」：回到导入页上传下一份文件
- 「下载失败行」（失败列表不为空时显示）：导出本次失败行 Excel，保留原始行号，方便修正后重导

### 5.4 操作反馈与过渡效果

所有异步操作必须防止用户中断导致数据不一致。

**全页面加载遮罩**：
- 提交导入/下载模板时，整个页面被半透明遮罩覆盖
- 遮罩上显示操作提示文字（如「正在提交审批，请勿关闭页面...」）
- 遮罩期间所有按钮和链接不可点击，页面不可操作

**按钮加载态**：

| 操作 | 遮罩文字 | 按钮效果 |
|------|---------|---------|
| 下载模板 | 「正在生成模板...」 | 按钮旋转加载图标 |
| 提交审批 | 「正在提交审批，请勿关闭页面...」 | 按钮旋转加载图标 |

**页面切换**：上传→预览→结果 三个状态使用淡入淡出+上下滑动过渡动画，避免生硬跳转。

### 5.5 审批集成

- 有效行自动提交审批，审批摘要由各类型自定义（行数、金额范围、涉及范围等）
- 审批通过 → 自动执行创建；驳回 → 数据保留可追溯

### 5.5 重导兼容

**场景覆盖**：

| 场景 | 处理方式 |
|------|----------|
| 格式校验部分行失败 | 失败行不进入审批，用户可下载失败行 Excel 修正后重新导入（新批次），**已审批通过的行不受影响** |
| 审批被驳回 | 整个批次标记 Rejected，用户可修正数据后重新导入（新批次），**新批次独立审批，旧批次数据仅保留审计追溯** |
| 审批通过后部分行重复 | 执行时二次检查唯一性，已存在的行自动跳过，**不阻塞新行的创建** |
| 同一份文件再次提交 | 允许，每次生成新 ImportBatch + 新审批流程，互不干扰 |

**兼容机制**：
- 每个导入批次独立 ID，互不影响
- 审批通过执行时的二次唯一性检查确保不会重复创建
- 已驳回的旧批次不会对新批次的校验和审批产生任何影响

### 5.6 撤回与重新提交

提交人可在审批待处理状态下撤回申请：

| 撤回方式 | 说明 |
|---------|------|
| 详情对话框「撤回」 | 审批详情底部显示「撤回」按钮（仅提交人可见） |
| 我的提交列表「撤回」 | 表格操作列直接点击撤回 |

撤回后：
- 审批请求状态 → `Cancelled`
- 关联的导入批次状态 → `Cancelled`（标记为可重新提交）
- 用户可回到导入页修正数据后重新上传，生成新批次+新审批流程

### 5.7 审批详情查看

点击审批列表的「详情」按钮可查看完整审批信息：

| 区块 | 内容 |
|------|------|
| **基本信息** | 审批类型、申请人、当前进度（第X/Y级）、审批角色、提交/完成时间、状态标签 |
| **审批流程步骤条** | 展示审批链各级角色及当前状态（completed/current/pending） |
| **关联数据** | 对导入类型的审批，显示文件名、行数统计、批次状态，可点击查看批次详情 |
| **审批记录时间线** | 每级审批人（头像+姓名）、操作（通过/驳回/提交）、级别、备注 |

### 5.8 导入数据可追溯

审批通过后，用户可通过以下路径查看最终导入的数据明细：

- **审批详情 → 关联数据 → 查看导入批次详情**
- 导入批次详情页展示所有行的完整数据：座楼/楼层/房号/完整编码/房型/面积/朝向/标准租金
- 支持按有效/失败筛选行
- 状态码统一显示中文：待审批、已通过、已驳回、已撤回

## 6. 房源导入特有需求

### 6.1 模板列

| 列名 | 必填 | 校验 |
|------|------|------|
| 座楼名称 | ✅ | 不可为空 |
| 座楼编码 | 可选 | — |
| 座楼地址 | 可选 | — |
| 楼层 | ✅ | 不可为空，提取数字作排序值 |
| 房号 | ✅ | 不可为空 |
| 房型 | ✅ | 必须匹配系统房型名称 |
| 面积(m²) | 可选 | >0 |
| 朝向 | 可选 | 仅允许预设值（东南西北等） |

### 6.2 金额自动计算

系统根据"楼层提取数字 → 匹配楼层分级 → 查询定价标准 → 赋值基础租金"，无匹配则标警告但不阻塞。

### 6.3 唯一性规则

`(座楼名称 + 楼层 + 房号)` 全局唯一，已存在的行在审批通过执行时自动跳过。

## 7. 非功能需求

- 单次上限 10000 行
- 1000 行以内 ≤ 5 秒返回
- 新增导入类型只需：定义扩展表 + 实现校验/执行逻辑 + 注册 DI
