# 项目命名规范

## API 路由

路由**不允许**包含连字符 `-`，统一使用全小写无分隔符格式。

### 正确示例

```
/api/housingunits          ✅
/api/housingunits/tree     ✅
/api/housingunits/buildinglist  ✅  （不要写 building-list）
/api/approvaltypes         ✅
/api/approvallevels        ✅
/api/approvals/pending     ✅  （pending 是单一路径段）
```

### 错误示例

```
/api/housing-units         ❌  含连字符
/api/housingunits/building-list  ❌  含连字符
/api/approval-types        ❌  含连字符
/api/payment-channels      ❌  含连字符
```

### 原理

`[controller]` 和 `[action]` 派生路由会自动从类名和方法名去掉连字符。自定义路由段也遵循此规则。

## 控制器命名

控制器类名使用单数或复数名词 + `Controller` 后缀，通过 `[Route("api/[controller]")]` 自动生成路由：

| 类名 | 路由 | 说明 |
|------|------|------|
| `HousingUnitsController` | `/api/housingunits` | 复数 |
| `ApprovalTypesController` | `/api/approvaltypes` | 复数 |
| `ApprovalsController` | `/api/approvals` | 复数 |

## 实体命名

- 类名使用 PascalCase
- 表名使用类名复数
- 命名空间保持简洁

## 前端路由

同样不允许连字符：

| 页面 | 路由 |
|------|------|
| 房源详情 | `/buildings/unit/:id` ✅ |
| 批量导入 | `/buildings/import` ✅ |
| 我的提交 | `/approvals/myrequests` ✅ （不要写 my-requests） |

## 前端 API 函数

函数名使用 camelCase，与服务端路由对应：

```javascript
// ✅ 正确
export function getHousingUnits(params)  // → GET /api/housingunits
export function getHousingUnitTree()      // → GET /api/housingunits/tree
export function getBuildingList()         // → GET /api/housingunits/buildinglist

// ❌ 错误
export function getHousingUnitsTree()     // 不要
export function getBuildingList()         // 函数名无连字符 OK，路由名也无连字符
```
